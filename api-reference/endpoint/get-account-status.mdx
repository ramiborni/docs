---
title: "Get Account Status"
openapi: "GET /accounts/status"
description: "Retrieve current status and usage limits for a LinkedIn account"
---

Query the health and capacity of a SendPilot-managed LinkedIn account. Use this endpoint to verify remaining quota and confirm whether an account is currently connected.

## Query Parameters

<ParamField query="linkedinAccountId" type="string" required>
  LinkedIn Account ID to check status for.
  
  Example: `acc_123456`
</ParamField>

## Response

<ResponseField name="accounts" type="array" required>
  Array of account status objects.
  
  <Expandable title="Account Status object">
    <ResponseField name="linkedinAccountId" type="string" required>
      LinkedIn Account identifier.
    </ResponseField>
    
    <ResponseField name="status" type="string" required>
      Account status (see status values below).
    </ResponseField>
    
    <ResponseField name="remainingConnectionRequests" type="integer" required>
      Remaining connection requests for today (minimum: 0).
    </ResponseField>
    
    <ResponseField name="remainingMessages" type="integer" required>
      Remaining messages for today (minimum: 0).
    </ResponseField>
    
    <ResponseField name="dailyConnectionCap" type="integer" required>
      Daily connection request limit.
    </ResponseField>
    
    <ResponseField name="dailyMessageCap" type="integer" required>
      Daily message limit.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash
curl -X GET "https://integration-api-gateway.agreeablesea-e4815929.swedencentral.azurecontainerapps.io/api/linkedin/accounts/status?linkedinAccountId=acc_123456" \
  -H "X-API-KEY: YOUR_API_KEY" \
```

## Account Status Values

| Status | Description | Can Send Messages |
|--------|-------------|-------------------|
| `CONNECTED` | Account is connected and operational | ✅ Yes |
| `DISCONNECTED` | Account session has expired or been invalidated | ❌ No |
| `PENDING_VERIFICATION` | Awaiting verification | ❌ No |
| `TWO_FA_REQUIRED` | Two-factor authentication required | ❌ No |
| `OTP_REQUIRED` | One-time password required | ❌ No |
| `CAPTCHA_REQUIRED` | CAPTCHA challenge required | ❌ No |
| `IN_APP_VALIDATION_REQUIRED` | In-app validation required | ❌ No |
| `PHONE_REGISTER_REQUIRED` | Phone registration required | ❌ No |
| `FAILED` | Connection failed | ❌ No |
| `EXPIRED` | Session expired | ❌ No |

## Usage Monitoring

Monitor your account status regularly to:
- **Track quota**: Ensure you have sufficient capacity for your campaigns
- **Prevent failures**: Avoid sending messages when quota is exhausted
- **Optimize timing**: Schedule campaigns based on quota availability
- **Monitor health**: Detect account connectivity issues early

## Best Practices

<Tip>
Check account status before launching campaigns to ensure sufficient quota is available.
</Tip>

- Poll account status before sending batches of connection requests or messages
- Implement quota-based throttling to stay within daily limits
- Set up alerts when remaining quota drops below a threshold
- Monitor `status` field to detect account disconnections
- Use status checks as part of your health monitoring

## Quota Management

### Daily Limits
LinkedIn accounts have daily limits that reset at midnight UTC:
- **Connection Requests**: Typically 50-100 per day (varies by account type)
- **Messages**: Typically 150-250 per day (varies by account type)

### Rate Limiting
In addition to daily caps:
- Spread requests throughout the day
- Avoid sending all messages in a short burst
- Implement delays between individual sends

## Workflow Example

```javascript
const axios = require('axios');

async function checkAccountStatus(linkedinAccountId) {
  const url = `https://integration-api-gateway.agreeablesea-e4815929.swedencentral.azurecontainerapps.io/api/linkedin/accounts/status?linkedinAccountId=${linkedinAccountId}`;

  const response = await axios.get(url, {
    headers: {
      'X-API-KEY': "YOUR_API_KEY",
      'Content-Type': 'application/json'
    }
  });

  return response.data.accounts[0];
}

// Check before sending campaign
async function sendCampaign(linkedinAccountId, leads) {
  const status = await checkAccountStatus(linkedinAccountId);
  
  if (status.status !== 'CONNECTED') {
    throw new Error(`Account is ${status.status}. Cannot send campaign.`);
  }
  
  if (status.remainingConnectionRequests < leads.length) {
    console.warn(`Insufficient quota. Only ${status.remainingConnectionRequests} requests remaining.`);
  }
  
  // Proceed with campaign...
}
```
