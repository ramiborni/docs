---
title: "Get Messages"
api: "GET /messaging/messages"
description: "Retrieve recent messages for reconciliation or webhook fallback scenarios"
---

Poll for recent LinkedIn messages across conversations. This endpoint is primarily used for:
- **Reconciliation**: Verify webhook delivery
- **Fallback**: Retrieve messages if webhooks are temporarily unavailable
- **Historical data**: Fetch conversation history

<Warning>
This endpoint is intended for occasional polling, not real-time message retrieval. Use webhooks for real-time message notifications.
</Warning>

## Query Parameters

<ParamField query="linkedinAccountId" type="string" required>
  LinkedIn account ID to retrieve messages for.
  
  Example: `clxxx123456`
</ParamField>

<ParamField query="limit" type="integer" required>
  Maximum number of messages to retrieve (minimum: 1).
  
  Example: `50`
</ParamField>

## Response

<ResponseField name="conversations" type="array" required>
  Array of conversations containing messages.
  
  <Expandable title="Conversation object">
    <ResponseField name="conversationId" type="string" required>
      Unique conversation identifier.
    </ResponseField>
    
    <ResponseField name="participants" type="array" required>
      Array of participants in the conversation (excluding the provided LinkedIn account).
      
      <Expandable title="Participant object">
        <ResponseField name="profileUrl" type="string" required>
          LinkedIn profile URL.
        </ResponseField>
        
        <ResponseField name="name" type="string" required>
          Full name of the participant.
        </ResponseField>
        
        <ResponseField name="profilePictureUrl" type="string">
          Profile picture URL.
        </ResponseField>
        
        <ResponseField name="headline" type="string">
          Professional headline.
        </ResponseField>
        
        <ResponseField name="location" type="string">
          Location.
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="messages" type="array" required>
      Array of messages in the conversation.
      
      <Expandable title="Message object">
        <ResponseField name="messageId" type="string" required>
          Unique message identifier.
        </ResponseField>
        
        <ResponseField name="senderProfileUrl" type="string" required>
          LinkedIn profile URL of the sender.
        </ResponseField>
        
        <ResponseField name="text" type="string" required>
          Message text content.
        </ResponseField>
        
        <ResponseField name="sentAt" type="string" required>
          Timestamp when message was sent (ISO-8601 format).
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="lastMessageAt" type="string" required>
      Timestamp of the last message in the conversation.
    </ResponseField>
    
    <ResponseField name="unreadCount" type="number" required>
      Number of unread messages in the conversation.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="page" type="integer" required>
  Current page number (starts at 0).
</ResponseField>

## Error Responses

### 400 Bad Request

Returned when query parameters are invalid:

```json
{
  "error": "Invalid query parameters",
  "message": "linkedinAccountId is required"
}
```

### 401 Unauthorized

Returned when API key is missing or invalid:

```json
{
  "error": "Invalid or missing API key"
}
```

## Best Practices

<Tip>
Use this endpoint for reconciliation and fallback scenarios. For real-time message notifications, use the Message Received webhook.
</Tip>

- Poll messages periodically (e.g., every 5-10 minutes) as a fallback mechanism
- Store `conversationId` and `messageId` to track message history
- Implement pagination using the `page` parameter for large result sets
- Use the `limit` parameter to control response size and API usage
- Monitor `unreadCount` to track new messages in each conversation
