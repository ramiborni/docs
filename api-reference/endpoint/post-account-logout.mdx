Trigger a managed account logout. Use this endpoint to invalidate the active LinkedIn session and disconnect the account from SendPilot.

## Request Body

<ParamField body="linkedinAccountId" type="string" required>
  LinkedIn Account ID to logout.
  
  Example: `acc_123456`
</ParamField>

## Response

<ResponseField name="action" type="string" required>
  Action performed. Returns `account_logged_out` when the account is successfully disconnected.
</ResponseField>

## Example Request

```bash
curl -X POST "https://integration-api-gateway.agreeablesea-e4815929.swedencentral.azurecontainerapps.io/api/linkedin/accounts/logout" \
  -H "X-API-KEY: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "linkedinAccountId": "acc_123456"
  }'
```

## Example Response

```json
{
  "action": "account_logged_out"
}
```

## When to Use Logout

Use the logout endpoint when you need to:
- **Disconnect an account**: Remove a LinkedIn account from active use
- **Security concerns**: Immediately invalidate sessions for security reasons
- **Account rotation**: Cycle through different accounts in your pool
- **Maintenance**: Temporarily disable an account for updates

<Warning>
Logging out an account will stop all active campaigns using that account. Messages and connection requests in progress may fail.
</Warning>

## Effects of Logout

When you logout an account:
1. The active LinkedIn session is invalidated
2. All scheduled messages and connection requests using this account are paused
3. The account status changes to "disconnected"
4. You'll receive an `account.disconnected` webhook event

## Reconnecting After Logout

After logout, you can reconnect the account using:
- [Reconnect LinkedIn Account](/api-reference/endpoint/post-account-reconnect-linkedin) - Restore access with updated credentials

## Common Errors

| Status | Error | Description |
|--------|-------|-------------|
| 200 | Success | Account logged out successfully |
| 400 | `Invalid request data` | The provided account ID is malformed or missing |
| 401 | `Invalid or missing API key` | API authentication failed |

## Workflow Example

```javascript
const axios = require('axios');

async function logoutAccount(linkedinAccountId) {
  const url = 'https://integration-api-gateway.agreeablesea-e4815929.swedencentral.azurecontainerapps.io/api/linkedin/accounts/logout';

  const response = await axios.post(url, {
    linkedinAccountId
  }, {
    headers: {
      'X-API-KEY': "YOUR_API_KEY",
      'Content-Type': 'application/json'
    }
  });

  console.log('Account logged out:', response.data.action);
  return response.data;
}

// Usage with campaign pause
async function safelyLogoutAccount(linkedinAccountId) {
  // Pause campaigns using this account
  await pauseCampaignsForAccount(linkedinAccountId);
  
  // Logout the account
  await logoutAccount(linkedinAccountId);
  
  // Update internal tracking
  await updateAccountStatus(linkedinAccountId, 'disconnected');
}
```

## Best Practices

<Tip>
Pause all active campaigns before logging out an account to prevent message delivery failures.
</Tip>

- Check for active campaigns before logout
- Implement graceful shutdown for pending operations
- Update your internal tracking immediately after logout
- Set up webhooks to monitor disconnection events
- Document logout reasons for future reference

## Account Lifecycle Management

### Planned Logout
For scheduled maintenance or rotation:
1. Check account status
2. Pause active campaigns
3. Wait for pending operations to complete
4. Execute logout
5. Update internal records

### Emergency Logout
For security incidents:
1. Execute logout immediately
2. Audit recent account activity
3. Update security protocols
4. Review and rotate credentials
5. Reconnect with enhanced security
