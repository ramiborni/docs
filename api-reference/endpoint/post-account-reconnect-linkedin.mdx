Refresh LinkedIn session cookies or credentials for an existing managed account. Use this endpoint when credentials expire or when an account becomes disconnected.

## Request Body

<ParamField body="linkedinAccountId" type="string" required>
  LinkedIn Account ID to reconnect.
  
  Example: `clxxx123456`
</ParamField>

<ParamField body="username" type="string" required>
  LinkedIn username - should be either an email address or a phone number.
  
  Example: `user@example.com`
</ParamField>

<ParamField body="password" type="string" required>
  LinkedIn account password.
  
  Example: `securepassword123`
</ParamField>

<ParamField body="userAgent" type="string" required>
  User agent string - if encountering disconnection issues, enter the exact user agent of the browser on which the account has been connected.
  
  Example: `Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36`
</ParamField>

<ParamField body="accessToken" type="string">
  LinkedIn access token for authentication.
  
  Example: `AQV8...`
</ParamField>

<ParamField body="premiumToken" type="string">
  LinkedIn premium token (if applicable).
  
  Example: `premium_token_123`
</ParamField>

<ParamField body="country" type="string">
  ISO 3166-1 A-2 country code (2 characters) to be set as proxy location.
  
  Example: `US`
</ParamField>

<ParamField body="ip" type="string">
  IPv4 address to infer proxy location.
  
  Example: `192.168.1.1`
</ParamField>

<ParamField body="proxy" type="object">
  Optional proxy configuration.

  <Expandable title="Proxy object">
    <ParamField body="protocol" type="string" required>
      Proxy protocol. Allowed values: `https`, `http`, `socks5`.
    </ParamField>
    
    <ParamField body="host" type="string" required>
      Proxy host address.
      
      Example: `proxy.example.com`
    </ParamField>
    
    <ParamField body="port" type="number" required>
      Proxy port number.
      
      Example: `8080`
    </ParamField>
    
    <ParamField body="username" type="string">
      Optional username for proxy authentication.
      
      Example: `proxyuser`
    </ParamField>
    
    <ParamField body="password" type="string">
      Optional password for proxy authentication.
      
      Example: `proxypass`
    </ParamField>
  </Expandable>
</ParamField>

## When to Use Reconnect

Use the reconnect endpoint when:
- **Credentials expired**: LinkedIn session has expired and needs refresh
- **Account disconnected**: Account was manually logged out or disconnected
- **Security checks**: LinkedIn requires re-authentication
- **Token refresh**: Access tokens need to be updated
- **Proxy changes**: You need to update proxy configuration

## Reconnect vs Connect

| Feature | Connect | Reconnect |
|---------|---------|-----------|
| Use case | New account onboarding | Existing account refresh |
| Account ID | Generated by API | Must be provided |
| Effect | Creates new account record | Updates existing record |
| Status | Creates `active` account | Changes to `active` from disconnected |

## Automatic Reconnection

Implement automatic reconnection for common scenarios:

```javascript
async function autoReconnect(linkedinAccountId, credentials) {
  try {
    const response = await fetch('https://integration-api-gateway.agreeablesea-e4815929.swedencentral.azurecontainerapps.io/api/account/accounts/reconnect-linkedin', {
      method: 'POST',
      headers: {
        'X-API-KEY': YOUR_API_KEY
  
  
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        linkedinAccountId,
        ...credentials
      })
    });

    if (!response.ok) {
      throw new Error(`Reconnect failed: ${response.statusText}`);
    }

    const result = await response.json();
    console.log('Account reconnected:', result.data.linkedinAccountId);
    return result;
  } catch (error) {
    console.error('Auto-reconnect failed:', error);
    // Alert team for manual intervention
    await alertTeam('reconnection_failed', linkedinAccountId);
    throw error;
  }
}
```

## Webhook Integration

Listen for disconnection events and automatically reconnect:

```javascript
// Handle account.disconnected webhook
app.post('/webhooks/account-status', async (req, res) => {
  const event = req.body;
  
  if (event.eventType === 'account.disconnected') {
    const { accountId, reason } = event.data;
    
    // Handle different disconnection reasons
    if (reason === 'invalid_credentials') {
      // Fetch stored credentials
      const credentials = await getStoredCredentials(accountId);
      
      // Attempt automatic reconnection
      try {
        await autoReconnect(accountId, credentials);
        console.log(`Auto-reconnected account ${accountId}`);
      } catch (error) {
        console.error(`Failed to auto-reconnect ${accountId}:`, error);
      }
    }
  }
  
  res.json({ received: true });
});
```

## Common Errors

| Status | Error | Description |
|--------|-------|-------------|
| 200 | Success | Account reconnected successfully |
| 400 | `Invalid request data` | Payload validation failures or account ID not found |
| 401 | `Invalid or missing API key` | LinkedIn rejected credentials or API authentication failed |
| 404 | `LinkedIn account not found` | The specified account ID doesn't exist |

## Best Practices

<Tip>
Store credentials securely and implement automatic reconnection for expired sessions to minimize downtime.
</Tip>

- Monitor for `account.disconnected` webhook events
- Implement automatic reconnection for `invalid_credentials` reason
- Use exponential backoff for retry attempts
- Keep user agent strings synchronized with actual browser
- Update credentials when reconnecting after security checks
- Test reconnection flow regularly
- Alert team when automatic reconnection fails

## Retry Strategy

Implement smart retry logic for reconnection failures:

```javascript
async function reconnectWithRetry(linkedinAccountId, credentials, maxRetries = 3) {
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      const result = await autoReconnect(linkedinAccountId, credentials);
      return result;
    } catch (error) {
      if (attempt === maxRetries) {
        throw new Error(`Failed after ${maxRetries} attempts: ${error.message}`);
      }
      
      // Exponential backoff
      const delay = Math.pow(2, attempt) * 1000;
      console.log(`Retry attempt ${attempt} after ${delay}ms`);
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
}
```
