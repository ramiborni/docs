---
title: "Send Connection Request"
api: "POST /messaging/connection-request"
description: "Send a LinkedIn connection request with an optional custom note"
---

Send a LinkedIn connection request to a target candidate. The request is queued and processed asynchronously by SendPilot's managed account infrastructure.

## Request Body

<ParamField body="leadUrl" type="string" required>
  LinkedIn profile URL of the lead.
  
  Example: `https://www.linkedin.com/in/john-doe`
</ParamField>

<ParamField body="linkedinAccountId" type="string" required>
  LinkedIn account database ID (use the `linkedinAccountId` returned from the connect LinkedIn endpoint).
  
  Example: `clxxx123456`
</ParamField>

<ParamField body="connectionNote" type="string">
  Optional connection note to include with the request (max 300 characters).
  
  Example: `Hi John, I noticed we share similar interests in tech...`
</ParamField>

## Response

<ResponseField name="action" type="string">
  Action performed. Returns `connection_request_sent` when the connection request is submitted.
</ResponseField>

## Example Request

```bash
curl -X POST https://integration-api-gateway.agreeablesea-e4815929.swedencentral.azurecontainerapps.io/api/linkedin/messaging/connection-request \
  -H "X-API-KEY: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "leadUrl": "https://www.linkedin.com/in/john-doe",
    "linkedinAccountId": "clxxx123456",
    "connectionNote": "Hi John, I'd love to connect and share more about our open role."
  }'
```

## Example Response

```json
{
  "action": "connection_request_sent"
}
```

## Best Practices

<Tip>
  **Personalize messages**: Connection requests with personalized messages have higher acceptance rates.
</Tip>

- Keep notes under 300 characters (LinkedIn's limit)
- Confirm the `linkedinAccountId` corresponds to a connected LinkedIn account before sending batches
- Validate LinkedIn URLs prior to submission to avoid malformed request errors
- Retry on transient infrastructure errors (503/504) with exponential backoff

## Connection Note Guidelines

LinkedIn has specific character limits for connection notes:
- **Premium accounts**: Up to 300 characters
- **Standard accounts**: Up to 300 characters (but shorter notes often perform better)

<Warning>
Connection notes are optional but highly recommended. Personalized connection requests have significantly higher acceptance rates.
</Warning>

## Common Errors

| Status | Error | Description |
|--------|-------|-------------|
| 200 | Success | Connection request sent successfully |
| 400 | `Invalid request data` | The payload is invalid, malformed, or the note exceeds allowed limits |
| 401 | `Invalid or missing API key` | API authentication failed |

## Workflow Example

```javascript
const axios = require('axios');

async function sendConnectionRequest(leadUrl, linkedinAccountId, note) {
  const url = 'https://integration-api-gateway.agreeablesea-e4815929.swedencentral.azurecontainerapps.io/api/linkedin/messaging/connection-request';

  const response = await axios.post(url, {
    leadUrl,
    linkedinAccountId,
    connectionNote: note
  }, {
    headers: {
      'X-API-KEY': "YOUR_API_KEY",
      'Content-Type': 'application/json'
    }
  });

  console.log('Connection request sent:', response.data.action);
  return response.data;
}
```
