Receive events when a previously invited LinkedIn contact accepts a connection request managed by SendPilot.

## Event Type

`CONNECTION_ACCEPTED` (or `connection.accepted`)

**Triggered when:** A LinkedIn connection request is accepted (30 seconds to 2 hours delay in sandbox)

## Webhook Event Payload

<ParamField body="eventId" type="string" required>
  Unique identifier for the webhook delivery. Use this for idempotency checks.
  
  Example: `evt_conn_accepted_1699358400000`
</ParamField>

<ParamField body="eventType" type="string" required>
  Always `connection.accepted` for this webhook.
</ParamField>

<ParamField body="timestamp" type="string" required>
  ISO-8601 timestamp when SendPilot detected the acceptance.
  
  Example: `2025-11-07T10:00:00.000Z`
</ParamField>

<ParamField body="data" type="object" required>
  Event data containing connection details.
  
  <Expandable title="Data object">
    <ParamField body="accountId" type="string" required>
      SendPilot-managed LinkedIn account that sent the original invitation.
      
      Example: `clxxx123456`
    </ParamField>
    
    <ParamField body="leadProfileUrl" type="string" required>
      LinkedIn profile URL of the lead who accepted the connection.
      
      Example: `https://www.linkedin.com/in/jane-smith`
    </ParamField>
    
    <ParamField body="connectionNote" type="string">
      The original connection note that was sent with the request.
      
      Example: `Hi Jane, I'd love to connect with you!`
    </ParamField>
    
    <ParamField body="originalRequestId" type="string" required>
      ID of the original connection request.
      
      Example: `req_abc123`
    </ParamField>
    
    <ParamField body="acceptedAt" type="string" required>
      ISO-8601 timestamp when the connection was accepted.
      
      Example: `2025-11-07T10:00:00.000Z`
    </ParamField>
  </Expandable>
</ParamField>

## Expected Response

Your webhook endpoint should respond with:

```json
{
  "received": true
}
```

**HTTP Status:** `200 OK`

## Handling the Webhook

1. **Verify signature** with the `X-SendPilot-Signature` header
2. **Enforce idempotency** using `eventId` before persisting the new connection
3. **Update CRM or ATS records** to reflect the newly connected relationship
4. **Trigger onboarding workflows** for the member if your automation requires next steps

<Warning>
Acknowledge receipt with `200 OK` after queuing the event for processing. Respond with `5xx` only when you need SendPilot to retry delivery.
</Warning>

## Implementation Example

```javascript
app.post('/webhooks/linkedin/account-accepted', async (req, res) => {
  const signature = req.headers['x-sendpilot-signature'];
  
  // 1. Verify webhook signature
  if (!verifyWebhookSignature(req.body, signature, WEBHOOK_SECRET)) {
    return res.status(401).json({ error: 'Invalid signature' });
  }
  
  const event = req.body;
  
  // 2. Check for duplicate events
  const isDuplicate = await checkEventProcessed(event.eventId);
  if (isDuplicate) {
    return res.status(200).json({ received: true });
  }
  
  // 3. Process the connection acceptance
  try {
    await db.connections.create({
      eventId: event.eventId,
      accountId: event.data.accountId,
      leadProfileUrl: event.data.leadProfileUrl,
      connectionNote: event.data.connectionNote,
      originalRequestId: event.data.originalRequestId,
      acceptedAt: new Date(event.data.acceptedAt)
    });
    
    // Update lead status in CRM
    await updateLeadStatus(event.data.leadProfileUrl, 'connected');
    
    // Trigger follow-up workflow
    await triggerFollowUpWorkflow(event.data.accountId, event.data.leadProfileUrl);
    
    // Mark event as processed
    await markEventProcessed(event.eventId);
    
    res.status(200).json({ received: true });
  } catch (error) {
    console.error('Error processing webhook:', error);
    res.status(500).json({ error: 'Processing failed' });
  }
});
```

## Retry Behavior

SendPilot implements automatic retries for failed webhook deliveries:

- **Retry on**: 5xx errors, network timeouts
- **No retry on**: 2xx, 4xx responses
- **Retry schedule**: Exponential backoff (1s, 5s, 30s, 5m, 30m)
- **Max attempts**: 5

## Best Practices

Use connection acceptance events to automatically trigger follow-up messages or update your lead scoring in real-time.
</Tip>

- Verify webhook signatures to ensure authenticity
