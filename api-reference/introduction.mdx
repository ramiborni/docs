The SendPilot LinkedIn Integration API enables you to programmatically send LinkedIn connection requests and messages, manage accounts, and receive real-time webhook notifications.

## Base URL

```
https://integration-api-gateway.agreeablesea-e4815929.swedencentral.azurecontainerapps.io/api/linkedin
```

## Endpoints

### Messaging
* **POST /messaging/connection-request**: Send LinkedIn connection requests to target candidates
* **POST /messaging/message**: Send LinkedIn messages to existing connections
* **GET /messaging/messages**: Poll for recent LinkedIn messages for reconciliation
* **GET /messaging/connection-status**: Check connection status with a LinkedIn profile

### Accounts
* **GET /accounts/status**: Retrieve health status, quotas, and limits for managed accounts
* **GET /accounts/linkedin**: List all LinkedIn accounts for your project
* **GET /accounts/linkedin/{id}**: Get details of a specific LinkedIn account
* **POST /accounts/connect-linkedin**: Connect a LinkedIn account to SendPilot
* **POST /accounts/connect-linkedin/checkpoint**: Solve LinkedIn checkpoint challenges (CAPTCHA, SMS, etc.)
* **POST /accounts/connect-linkedin/checkpoint/resend**: Resend checkpoint verification code
* **POST /accounts/reconnect-linkedin**: Reconnect an existing LinkedIn account
* **POST /accounts/logout**: Logout and disconnect a LinkedIn account
* **DELETE /accounts/linkedin/{id}**: Permanently delete a LinkedIn account

### Webhooks Management
* **POST /webhooks/register**: Register a webhook URL to receive event notifications
* **DELETE /webhooks/unregister**: Remove a webhook URL from receiving events
* **GET /webhooks/list**: List all registered webhooks for your account

### Webhook Events
* **message.received**: Inbound LinkedIn message received
* **message.sent**: Outbound message successfully delivered
* **connection.accepted**: Connection request accepted
* **connection.rejected**: Connection request rejected
* **account.connected**: Account connected to SendPilot
* **account.disconnected**: Account disconnected
* **account.deleted**: Account permanently deleted
* **profile.viewed**: Profile viewed through SendPilot

## Authentication

All API requests require authentication using an API key header. Include the following header in every request:

```bash
X-API-KEY: YOUR_API_KEY
```

### Example Request

```bash
curl -X POST https://integration-api-gateway.agreeablesea-e4815929.swedencentral.azurecontainerapps.io/api/linkedin/messaging/message \
  -H "X-API-KEY: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "linkedinAccountId": "clxxx123456",
    "leadUrl": "https://www.linkedin.com/in/john-doe",
    "text": "Hi John, I wanted to reach out about an opportunity."
  }'
```

<Warning>
Keep your authentication credentials secret. Never commit them to version control or expose them in client-side code.
</Warning>

## Environments

SendPilot supports different environments through separate API keys:

| Environment | Description | API Key Format |
|------------|-------------|----------------|
| Sandbox | Testing and development | Separate sandbox API key |
| Production | Live production traffic | Production API key |

Use the appropriate API key for your target environment.

## Error Handling

The API uses standard HTTP status codes and returns structured error responses:

```json
{
  "error": "Invalid request data",
  "message": "linkedinAccountId is required"
}
```

Common status codes:
- `200`: Success
- `201`: Created (message sent)
- `202`: Accepted (async processing)
- `400`: Bad request (invalid payload)
- `401`: Unauthorized (invalid credentials)
- `404`: Not found
- `429`: Rate limit exceeded
- `500`: Internal server error

## Rate Limits

API rate limits vary by endpoint and environment:

- **Sandbox**: More permissive limits for testing
- **Production**: Stricter limits to protect account health

Monitor your rate limits using the [Account Status endpoint](/api-reference/endpoint/get-account-status).

## Webhooks

SendPilot can send real-time notifications to your webhook endpoints for various events:

**Messaging Events:**
- **Message Received**: When a LinkedIn message arrives
- **Message Sent**: When an outbound message is delivered

**Connection Events:**
- **Connection Accepted**: When a connection request is accepted
- **Connection Rejected**: When a connection request is declined

**Account Events:**
- **Account Connected**: When an account connects to SendPilot
- **Account Disconnected**: When an account loses connectivity
- **Account Deleted**: When an account is permanently removed

**Activity Events:**
- **Profile Viewed**: When a profile is viewed through SendPilot

Register webhooks using the [Webhook Registration endpoint](/api-reference/endpoint/post-webhooks-register).

## Getting Started

1. **Obtain API key**: Get your API key from SendPilot
2. **Connect a LinkedIn account**: Use the [Connect LinkedIn endpoint](/api-reference/endpoint/post-account-connect-linkedin)
3. **Send your first message**: Try the [Send Message endpoint](/api-reference/endpoint/post-message)
4. **Set up webhooks**: Register webhooks to receive real-time notifications

## SDK & Libraries

Official SDKs coming soon. For now, use standard HTTP clients:

```javascript
// Node.js example
const sendMessage = async (linkedinAccountId, leadUrl, text) => {
  const response = await fetch('https://integration-api-gateway.agreeablesea-e4815929.swedencentral.azurecontainerapps.io/api/linkedin/messaging/message', {
    method: 'POST',
    headers: {
      'X-API-KEY': process.env.SENDPILOT_API_KEY,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ linkedinAccountId, leadUrl, text })
  });
  
  return response.json();
};
```
